# Chile MORA line — evidence status (Helia's Pérez branch)

Single place to see **what is proven**, **what is indexed-only**, **what conflicts**, and **what must not be merged** without a record.

## 1. Tree rule (operational)

**`family-tree.json` only gets parent–child edges that a primary record supports for that exact person** — baptism, marriage intent with filiation, burial with usable ages/filiation, or civil registration that names the link.

Hypotheses (Marilao → Bartola → Isabel, Callapan / Millacoyan context, thematic indigenous analysis, "this Isabel is Mapuche because…") belong in **topics**, **sources**, and **prose**, not as descent lines in JSON until a record names the relationship.

This file is the checklist so "no fake lines" stays operational, not just intent.
