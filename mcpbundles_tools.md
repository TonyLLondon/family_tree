### `upsert_bundle`

Manage bundles: enable/disable access, bind credentials, update settings (memory, read_only_mode per provider), create composed bundles from template bundles, or modify bundle composition. Provide bundle_slug and only the fields you want to change. Use credential_bindings to explicitly attach credentials when auto-bind is ambiguous.

**Parameters:**

- `bundle_slug` (string, **required**): Bundle slug to update. Required. Example: 'attio-crm', 'pdl-people'. If the slug does not exist and 'name' is provided, a new composed bundle is created.
- `enabled` (boolean, optional): Enable or disable the bundle in the workspace. Set to true to activate the bundle (credentials auto-bind if available). Set to false to deactivate it (access record is kept). Omit to leave the current access status unchanged.
- `name` (string, optional): Display name for a new composed bundle. Only used when creating a new bundle (when bundle_slug does not exist). Example: 'Enrichment Pipeline'.
- `description` (string, optional): Description for a new composed bundle. Only used when creating.
- `template_slugs` (array, optional): List of bundle slugs to compose into this bundle. Each slug becomes a template source — the composed bundle inherits all their tools at read time. This replaces any existing composition. Example: ['attio-crm', 'pdl-people']. Only applicable to user-owned (composed) bundles.
- `memory` (object, optional): The memory object to persist. Must be a valid JSON object (not a string). This fully replaces any existing memory — it is NOT a partial merge. Maximum size: 64 KB. Pass an empty object {} to clear memory.
- `memory_enabled` (boolean, optional): Enable or disable persistent memory for this bundle. When enabled, bundle_get_memory/bundle_set_memory tools appear in the bundle's tool list.
- `read_only_mode` (boolean, optional): Set read-only mode for ALL providers in this bundle. When enabled, only tools with readOnlyHint=true are visible on the bundle's MCP endpoint. This writes through to all credential bindings.
- `credential_bindings` (array, optional): Bind specific credentials to this bundle for each provider. Each entry maps a provider_slug to a credential_id (UUID). Use list_credentials to find available credential IDs. This resolves ambiguity when multiple credentials exist for the same provider.

### `upsert_credential`

Create or update a provider credential. For FIELDS auth, pass credential_fields with the provider's required fields (e.g. api_key). For OAUTH2, opens the MCPBundles dashboard in a browser to complete the OAuth flow. Auto-validates and auto-binds to the provider's primary bundle on success.

**Parameters:**

- `provider_slug` (string, **required**): Provider slug (e.g. 'hubspot', 'stripe', 'brevo'). Required.
- `credential_id` (string, optional): Existing credential UUID to update. Omit to create a new credential.
- `credential_name` (string, optional): Human-friendly name for the credential (e.g. 'Production API Key').
- `credential_fields` (object, optional): Credential field values as key-value pairs. For FIELDS auth: the provider's required fields (e.g. {"api_key": "xkeysib-xxx"}). Required for FIELDS create. For update, only include fields to change (merged with existing).
- `bundle_slug` (string, optional): Target bundle slug for auto-binding after creation. If omitted, binds to the provider's primary bundle.
- `auth_type` (string, optional), one of: `['FIELDS', 'NONE', 'OAUTH2']`: Authentication type. Default: FIELDS. Use OAUTH2 to open the MCPBundles dashboard where the OAuth flow can be completed in a browser.

### `upsert_provider`

Add a remote MCP server to your workspace. Analyzes the server URL to detect transport, tools, and auth requirements. For servers requiring an API key, pass api_key. For OAuth servers, opens the MCPBundles dashboard in a browser to complete the OAuth flow. Creates the provider, bundle, credential, and tool discovery in one step.

**Parameters:**

- `server_url` (string, **required**): MCP server URL (e.g. 'https://mcp.example.com/sse'). The server will be analyzed to detect transport, tools, and auth requirements.
- `api_key` (string, optional): API key or access token for servers that require FIELDS authentication. Sent as a Bearer token during analysis and stored as the credential.
- `name` (string, optional): Display name for the provider. Auto-detected from server info if omitted.
- `compose_into_bundle_slug` (string, optional): Slug of an existing bundle to compose this server into. If omitted, a new bundle is created.
